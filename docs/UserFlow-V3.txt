4. SMART REVIEW QUEUE (Key Screen)
================================================================================

Screen: Focus-Based Review Queue
Route: /v3/review-queue (or tab within transactions)

Header:
  - "Action Required" label
  - "Review Queue" headline
  - "8 Items Needing Attention" subtitle
  - "Finish & Update Dashboard" CTA button

Layout: 12-column bento grid

Section 1 — Uncategorized Transactions (8 cols):
  - Grid of cards (2 columns)
  - Each card shows:
    - Merchant name + date + amount
    - Contextual icon (coffee, cloud, car, etc.)
    - Category dropdown selector
    - "Save as Auto-Rule" toggle switch
  - Only shows transactions that need action

Section 2 — AI Confidence Review (4 cols):
  - Suggestion cards for low-confidence categorizations
  - Each shows:
    - Merchant name
    - "Suggestion" or "Learned Rule" badge
    - Suggested category with explanation
    - Confirm / Edit buttons
  - Shows when system has a guess but isn't confident

Section 3 — Possible Duplicates (12 cols, bottom):
  - Side-by-side comparison cards
  - Each duplicate group shows:
    - Merchant name
    - "Potential Double Charge" or "Similar Activity" badge
    - Transaction A vs Transaction B
    - Amount, date, time for each
    - "Keep This" / "Discard" buttons

Floating FAB:
  - "Finish Review" button (bottom-right)
  - Tooltip on hover

Key UX principles:
  - ONLY show what needs attention
  - Don't force users to review 200 good transactions
  - Make categorization feel like a conversation, not a chore

KEY UX PRINCIPLES
================================================================================

  1. Don't force users to review everything
     → Only show what matters

  2. Make it feel smarter over time
     → Rules = secret weapon. App learns from every correction.

  3. Fast "time to value"
     → User should get insights in < 2 minutes

  4. Trust > automation
     → Always let user override system

  5. The ongoing loop
     → Each month is faster because rules accumulate
